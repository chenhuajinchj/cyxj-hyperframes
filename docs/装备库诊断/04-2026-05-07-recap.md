### 🔴 Level 1: Token CSS 抽离（**三选一未决**）

**问题**：每个新工程要手抄 `:root` CSS 变量（颜色/字号/字体三件套）；DNA 修订（如这次 2026-05-06 深蓝→暖米）时要挨个改各文件。

**抽法**：建 `templates/components/xcyj-tokens/xcyj-tokens.css`，把 DNA 全部 token 定义在里面。新工程一行 `<link>` 引用：

```html
<link rel="stylesheet" href="../templates/components/xcyj-tokens/xcyj-tokens.css">
```

**收益**：DNA 文档 → 真正的可执行代码；DNA 改了所有用此 token 的工程自动跟上。

## 4. 讨论但未行动（教学 / 知识更新 / 监控）

### 4.1 Tailwind vs 模块化（教学讨论）

用户疑惑：Tailwind 是不是跟"模块化"如出一辙？

**结论**：Tailwind 是**原子级**模块化（`.bg-red-500 .p-4`，每个 class 管 1 个属性），视频生产需要**组件级**模块化（cc-window 一个 class 管整组件）。

**金字塔**：
```
Level 4: 完整 Tailwind 工具链              ❌ 不建议
Level 3: 自建 utility class 库（.bg-hot 等）  ❌ 不建议
Level 2: 复用组件（cc-window 那样）          ✅ 在自然演化
Level 1: Token CSS 文件（颜色/字号/字体）    ⭐ 见上节"未决"
```

**Tailwind 真正的应用场景**：
- SaaS Dashboard / 后台 / 落地页（Vercel / Linear / Stripe docs）
- 设计师快速 mock UI 试样式
- 多人协作 Web 项目避免 class 冲突
- 短期项目快速上线

**不适合**：视频生产 / 强品牌产品 / 需要长期沉淀视觉资产 / 移动 App 原生

**类比闭环**：Tailwind = 乐高散装零件（自由拼），cc-window 体系 = 模型套件（固定的"宇航员套装"）。视频要的是后者。

- `/remote-control`（短形 `/rc`）—— 生成 QR 码 / 会话链接，手机 app 扫码连

- 部分命令（`/clear` / `/compact`）从手机不完全 work
