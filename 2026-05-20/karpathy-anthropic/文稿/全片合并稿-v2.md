## Ch0 — 开场（~30s）

2026 年 5 月 19 号，下午 3 点 05 分，Karpathy 发了一条推文。

就一句话："Personal update: I've joined Anthropic."

就这十个字，把整个 AI 圈炸了。

因为这个人是谁？他是 OpenAI 的创始成员。而 Anthropic——也就是 A 社——上个月刚刚第一次反超了 OpenAI。

一个 OpenAI 的创始成员，跑去了刚反超 OpenAI 的对手。

为什么是 A 社？为什么是现在？

在回答这两个问题之前，先认识一下这个人。

## Ch1 — 这个人是谁（~75s）

他叫 Andrej Karpathy。如果你在 AI 圈，这个名字你肯定听过。如果你刚入门，我用一句话告诉你他是谁——

现代 AI 教学的 GOAT 之一，OpenAI 创始成员，特斯拉 AI 总监，离开，再回 OpenAI，再走，做了个 AI 教育公司，现在去了 A 社。

这人有意思就有意思在这个轨迹上。一站一站来看。

**第一站**：多伦多大学，2005 到 2009 年，计算机加物理双学位。在那旁听了 Geoff Hinton 的课——就是后来拿诺贝尔奖的 Hinton。

**第二站**：英属哥伦比亚读硕士，2009 到 2011，方向是怎么让机器人靠机器学习学会走路跑步。

**第三站**：斯坦福博士，2011 到 2015，导师是李飞飞，做视觉和语言交叉。期间在谷歌实习过两次。

**第四站**：2015 年，OpenAI 成立，他是创始成员。注意是**创始成员**——不是 Sam Altman 或 Musk 那个级别的联合创始人，但他名字在 2015 年的创始公告里。这一段他还在斯坦福开了一门课，CS231n，讲卷积网络，第一年 150 个学生，两年后 750 个。

**第五站**：2017 年 6 月，去了特斯拉，直接向马斯克汇报，做了五年多到 2022 年 7 月离开。

**第六站**：2023 年 2 月，他回了 OpenAI，建了一个新团队。刚好一年，2024 年 2 月又走了。

**第七站**：2024 年 7 月，他创立 Eureka Labs，做 AI 教育——原话是"如果成功，让任何人都能学会任何东西"。但随着加入 A 社，这个项目目前暂停。他说以后会继续做。

**第八站**：就是上周——加入了 A 社的预训练部门，A 社还专门为他新建了一个子团队，专注研究**怎么用 Claude 加速训练下一代 Claude**。

注意：他**两次**离开 OpenAI——第一次去特斯拉，第二次去做教育。这本身就有戏。

但他真正这两年出圈的，不只是这个履历——是他提的两个词。

## Ch2 — 出圈的两个词（~55s）

我们倒着讲，从最新的讲。

**第一个：vibe coding。**

这个词是他自己造的，2025 年 2 月 2 号一条推文。原话是 "give in to the vibes"——投降给氛围。意思就是：完全相信 AI 写的代码，人不去看代码本身，只看效果对不对。

这条推文当时他发完自己都没当回事。结果一年下来，这个词彻底出圈了，到现在 vibe coding 已经变成一种泛指的工作方式——你说一句话，AI 写代码，你看效果。

**第二个：LLM Wiki。**

时间往前一个月，2026 年 4 月 4 号，他在 GitHub 上发了一个 Gist——不是独立项目，就是一个公开的笔记——叫 LLM Wiki。

核心想法是这样的：现在大家都在用 RAG，每次问问题就跑去外部资料库捞一遍，他说这有点浪费。不如建一个**由 AI 来持续维护的知识库**——你的笔记、文档、SOP 分成 raw、wiki、schema 三层放好，让 agent 帮你持续整理。知识可以越攒越多，不用每次从零捡。这个 Gist 现在 5000 多个 star。

**收束**

你看出门道没有？他这两年做的事，**没有一件是去训练新 model 的**。全是在做：**怎么让 model 用得更好**。

这恰好就是 A 社这一年在做的事。

## Ch3 — A 社的势头（~75s）

先看一个数据。

2023 年 6 月，A 社在企业 AI 市场的采纳率是多少？**0.03%**。

不是我少打一个零——就是 0.03%，几乎可以忽略。

2025 年 4 月——7.94%。

2026 年 4 月——**34.44%**。同一个月，OpenAI 是 32.3%。**A 社首次超过 OpenAI。**

三年从 0.03% 到 34%，而且是在 OpenAI 峰值 36.5% 之后追上来的。

这个数据来自 Ramp AI Index，覆盖了 5 万家美国企业的真实消费记录。要说清楚，这是 Ramp 的客户群，不代表整个市场——OpenAI 还有大量大客户合同和消费端的盘子。但作为势头信号，很难忽视。

第二件事，今年 5 月 4 号，A 社宣布联合 Blackstone、高盛等几家机构，一起办了一家新的企业 AI 服务公司。

目标客户不是科技大厂——是社区银行、中型制造商、区域医疗机构。

A 社不只是在出 model，是在出一整套**怎么用 model**的方法——工程师、服务层、合作网络，全都跟着走。

这就是 Karpathy 和 A 社对上眼的关键——他们在同一个理念上，已经平行走了一年。

## Ch4 — 外壳才是产品（~110s）

这个理念有个名字，叫 **wrapper**——翻成中文就是"外壳"。

我下面全用"外壳"。

大多数人讨论 AI，还是从 model 出发：GPT 好还是 Claude 好，Gemini 上了哪个榜，跑分谁高。Model 当然重要，不是说它不重要——但如果你真用过这些工具一段时间，你会注意到一件事：**同一个 model，不同人用，结果差得离谱**。

那个差距在哪里？在外壳上。

什么叫外壳？把 model 放在中心——Claude 也好，GPT 也好，Gemini 也好——中心就是一个模型。然后往外一层一层套：

- 第一层是 Claude Code、Codex，是命令行工具
- 第二层是 Skills、Subagents，是功能扩展
- 第三层是 Hooks、MCP，是事件和协议
- 最外面是 Memory 和 CLAUDE.md，是你给 model 的上下文环境——它知道你是谁、你在做什么、你要什么样的结果

这整个洋葱，从里到外，才是你**真正在用的产品**。

Model 不是护城河了。**外壳才是产品**。

我自己用 Claude Code 两个月，最深的体感就是这个。Model 每次升级，感觉就强一点点，从 100 分提到 105 分。但是把 CLAUDE.md 写好、把 skill 配对、把任务拆细——同一个 Claude，我用出来的效果比刚开始能强一倍。

**你不是在等更好的 model，你是在学怎么更好地用这个 model**。

这也是为什么 Karpathy 加入 A 社这件事值得认真看一眼——不只是"大牛跳槽"。

他这次加入 A 社，**不是来训练新 model 的**。A 社官方说的是——他要在预训练团队下，**组建一个新的子团队**，专门研究**用 Claude 加速训练下一代 Claude**。

就这一句话，绕了一个圈。

**用现在的 Claude 这层外壳，去帮训练下一代 Claude。**

外壳思维落到极致，就长这个样子。

## Ch5 — 两条河汇流（~85s）

所以如果你把 Karpathy 这两年做的事，和 A 社这一年做的事并排放——

你会发现这是两条已经汇流的河。

左边这条河是 Karpathy。右边这条河是 A 社。把它们并排放，不是巧合，是**同一个方向**。

我说 3 对。

**第一对：LLM Wiki 和 Claude memory。**

Karpathy 的 LLM Wiki，想法是让 AI 帮你把笔记、文档、SOP 整理成一个会生长的知识库——你的小数据，时间越长越有价值，换 model 也不带走，留在你这里。

A 社这边在做什么？Claude 的项目 memory、skill 文件、CLAUDE.md——都是同一件事：**把你的工作方式写死在上下文里，让 model 越来越懂你这个人**。

合流点：**你的小数据成为护城河**。不是 model 锁死你，是你的上下文舍不得重来。

**第二对：vibe coding 和 Claude Code。**

vibe coding 是什么？用自然语言描述你要什么，AI 写代码，你坐那边看、改方向、说感觉对不对。Karpathy 2025 年 2 月造的词，当时很多人觉得是玩笑——但这两年用过 Claude Code 的人都知道，**这就是日常工作流**。

Claude Code 就是 vibe coding 最成熟的那个实现。**你说，它做。**

合流点：**同一种用法，只是终于有了像样的工具**。

**第三对：Eureka Labs 和 A 社缺的最后一块。**

这一对是我觉得最有意思的。Karpathy 在加入 A 社的推文里，专门写了一句话——他对教育依然 deeply passionate，"plan to resume my work on it in time"。注意是 resume，恢复——意思是现在暂停了，不是结束了。

Eureka Labs 目前是暂停状态。但**他把教育这件事专门作为加入 A 社时的特别说明带进去了**。

A 社缺什么？他们有模型，有 Claude Code，有企业服务——**但非开发者怎么用 Claude，这套东西还没有**。

而 Karpathy 最擅长的事，就是把复杂的 AI 系统讲得让普通人能用、能理解、能上手。

合流点：**教育层**。这一块是 Karpathy 带过去的，A 社原本就缺。

**收束**

我的感受是这样的：Karpathy 这两年所有的实验和教学，其实都在做一件事——**让普通人能驾驭 model**。A 社这一年所有的产品，也在做这件事。

两条河迟早会汇到一起，区别只是在哪家公司里。

这条河汇了之后会流向哪？我猜 3 个方向——

## Ch6 — 3 个预测（~95s）

先说清楚，这是猜——不是 A 社的路线图，也没有内部消息。但顺着前面这条逻辑往下推，应该往这 3 个方向走。

**预测一：上下文市场。**

我猜 A 社会做一个"上下文集市"——但不是那种"买一个写营销文案的 prompt 模板"，那种东西不值钱。

我说的是更深的东西：**skills + workflows + project memories + 评测循环，打包成一个上下文包，放进一个市场里**，让人按需接入。

举个例子：一家会计事务所，把他们的月结流程、审计 SOP、常见错误模式，全部打包上架——另一家小公司的财务直接接进来，Claude 就知道怎么帮他跑月结，不用从零开始解释流程。

A 社已经在做 Skills 这条路了，往集市的方向延伸是顺理成章的。

而且这一块 Codex 那边已经走在前面，A 社不跟也得跟。

**预测二：/goal 风格的命令家族。**

现在 Claude Code 有 /goal——给 agent 一个目标，让它一直工作到你说停。这个命令最初是 OpenAI Codex 团队的 Eric Traut 做的，不是 A 社原创——但这个接口逻辑太对了，所以大家都在跟进。

接下来我猜会出现的：/research——让 agent 自主搜集资料、对比来源、出报告。/debug——给一个 bug 描述，agent 自己跑测试、找根因、改到绿灯。/migrate——告诉 agent "帮我把这个代码库从 v1 迁到 v2"，它自己去啃文档、改代码、跑测试、提 PR。

**接口的根本变化只有一个：从"做这一步"，变成"达成这个状态"**。你不再管它怎么做，你只管它做完没有。

**预测三：教育层。**

这是我觉得最有趣的猜测。

Karpathy 推文里专门写了 "resume my work on education in time"。他把教育放进来了，而且说 resume，说明这事在他心里没有结束，只是暂停。

所以我猜 Eureka Labs 那条线，会在 A 社内部以某种形式重生。不一定叫同一个名字，但方向应该是：**让非开发者，把脑袋里的领域 know-how 打包成 agent**。

举个例子：会计师懂月结流程，这个知识目前在他脑袋里。地产经纪懂客户接待 SOP，这个在他的经验和直觉里。YouTuber 懂什么选题有流量，这些在他脑子里，不在任何文档里。

这些人不是开发者，让他们写 CLAUDE.md 是强人所难。但如果有一套足够简单的打包工具，让他们用自然语言描述流程、上传样本、教 agent 什么叫"好"——然后这个 agent 可以上架、可以共享、可以给同行用——

**这才是真正的"每个人都能用 Claude"**。

这一条最没把握，但也最想看到。

## Ch7 — 收尾（~50s）

最后两句话。

第一句：**外壳才是产品**这个 thesis，最近其实越来越明显。

你看昨天谷歌大会，他们之前出了一个叫 Antigravity 的 IDE，这一次大会直接砍掉了，做成了类似 Codex 这样的**超级应用 App**。

超级 App 就是未来的趋势——而 Claude Code 自己的桌面端，**用过的人都知道**。Codex 也好，Claude Code 也好，**真正的差距早就不在 model 上了**。

第二句：Karpathy 加入 A 社这件事，可能就是这个 thesis 的最强信号。
